# Function 6: Compare and Merge Seeklight CSV

This function compares Seeklight-transformed metadata CSV files with your core metadata CSV and allows selective merging of changes.

## Purpose

After transforming Seeklight-generated metadata using Function 5, Function 6 helps you:
- Compare the transformed Seeklight metadata with your core metadata CSV
- Review field-by-field changes before merging
- Selectively merge only the changes you want to keep
- Preserve your core metadata structure while incorporating AI-generated metadata

## Workflow

### 1. Prerequisites
- Core metadata CSV must be configured in Function 0 (App Settings)
- One or more Seeklight-transformed CSV files must exist (generated by Function 5)
  - Files named: `DART_seeklight_transformed_YYYYMMDD_HHMMSS.csv`
  - Located in: `.DART-working-directory` subfolder

### 2. Running the Comparison

1. **Click Function 6** button
2. **Select CSV** (if multiple): The newest Seeklight CSV is auto-selected and highlighted
3. **View Results**: Shows comparison summary:
   - Records with no changes (matched)
   - New records (in Seeklight but not in core)
   - Records with field changes

### 3. Review and Merge

Click **"Review & Merge"** to see an interactive merge interface:

#### Merge Instructions
- **CHECKED**: This change will be merged into the core CSV (accept the Seeklight value)
- **UNCHECKED**: This change will be skipped (keep the existing core value)

#### Display Format
All field changes are shown in this format:
- **`CORE (old) ← SEEKLIGHT (new)`**
- The **arrow points LEFT** (←) meaning "replace core value with Seeklight value"
- **Left side**: Current value in your core metadata (what you have now)
- **Right side**: New value from Seeklight AI (what will replace it if checked)
- Example: `dc_title: "Untitled" ← "Portrait of a Woman"` means replace "Untitled" with "Portrait of a Woman"

**Protected Fields**: The following fields are never shown as changeable:
- `objectid` - Used for matching, never modified
- `filename` - Used for matching, never modified  
- `filepath` - Internal DART field

#### Change Categories

**✨ NEW RECORDS** (blue)
- Records present in Seeklight CSV but not in core metadata
- Each record has a checkbox to add it to core
- Default: All checked (ready to add)

**📝 CHANGED RECORDS** (orange)
- Records that exist in both files but have different field values
- Each field has its own checkbox for granular control
- Shows: `field_name: old_value → new_value`

**⚠️ DATA LOSS WARNINGS** (orange background)
- Fields where Seeklight value is empty but core has data
- Default: UNCHECKED to prevent accidental data loss
- Requires manual review before enabling

### 4. Execute Merge

1. **Review selections**: Check/uncheck individual fields as needed
2. **Click "Merge Selected"**: Applies only the checked changes
3. **Automatic backup**: Creates timestamped backup of core CSV
   - Format: `{core_csv_name}.backup_YYYYMMDD_HHMMSS`
4. **Core CSV updated**: Changes are written to your core metadata CSV

## Matching Logic

Function 6 uses **basename-to-objectid matching**:
- Seeklight CSV `filename` basename (without extension) ↔ Core CSV `objectid`
- Example: Seeklight filename "image001.jpg" → basename "image001" → matches core objectid "image001"
- This differs from Function 4 which matches `filename` to `filename`

**How it works:**
1. For each row in the Seeklight transformed CSV:
   - Extract the `filename` field (e.g., "image001.jpg")
   - Get the basename without extension (e.g., "image001")
   - Look for a core CSV row where `objectid` equals that basename
2. If match found: Compare field values and track changes
3. If no match: Mark as new record to add

This matching method assumes your core metadata uses the file basename as the objectid (common in many CollectionBuilder projects).

## Best Practices

### Before Merging
- ✅ Review field changes carefully, especially title fields
- ✅ Pay attention to data loss warnings (empty Seeklight values)
- ✅ Verify new records before adding them
- ✅ Use Function 1 first to ensure all files have objectids

### After Merging
- ✅ Check the backup file location (shown in logs)
- ✅ Test the merged CSV with your CollectionBuilder site
- ✅ Keep backup files for rollback if needed

### Selective Merging
- Merge Seeklight titles separately from other fields
- Add new records first, then review field changes
- Use multiple passes for large batches (merge in stages)

## Comparison with Function 4

| Feature | Function 4 | Function 6 |
|---------|-----------|-----------|
| **Input Files** | DART_export_*.csv | DART_seeklight_transformed_*.csv |
| **Matching** | filename to filename | filename basename to objectid |
| **Purpose** | Merge new asset batches | Merge AI-generated metadata |
| **Source** | Function 2 exports | Function 5 transformations |
| **Compound Objects** | Full support with parent/child | Standard field-level |

## Tips

1. **Incremental Merging**: Process small batches of Seeklight data at a time
2. **Title Fields**: Often require manual review - Seeklight may generate generic titles
3. **Multiple Runs**: You can run Function 6 multiple times with different selections
4. **Backup Management**: Keep recent backups; delete old ones to save space
5. **Field Mapping**: Edit `seeklight_mapping_template.json` to control which Seeklight fields map to core columns

## Error Prevention

- **Always backup**: Function automatically creates backups before merge
- **Data loss protection**: Empty Seeklight values are unchecked by default
- **Selective control**: Uncheck any field you don't want to change
- **Cancel option**: Close dialog without merging if you change your mind

## Output

- **Updated core CSV**: Your core metadata CSV with merged changes
- **Backup file**: Timestamped backup of original core CSV
- **Log entries**: Detailed record of what was merged

## Troubleshooting

**No Seeklight CSV files found**
- Run Function 5 first to transform Seeklight metadata
- Check `.DART-working-directory` subfolder for DART_seeklight_transformed_*.csv files

**Records don't match / All marked as "new"**
- Check that your core CSV `objectid` values are the file basenames (without extensions)
- Example: For file "image001.jpg", core CSV should have objectid="image001"
- Verify Seeklight CSV has correct filenames in the `filename` column
- Check for case sensitivity differences (Function 6 is case-sensitive)

**Fields not appearing**
- Edit `seeklight_mapping_template.json` to add field mappings
- Re-run Function 5 to regenerate transformed CSV with new mappings
